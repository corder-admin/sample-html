Section 01

背景と課題

建設業界における見積書作成は、過去案件のExcelファイルを参照しながら手作業で単価を調整する属人的な業務であり、担当者の経験と勘に大きく依存しています。

業界が直面している主要課題は以下の通りです。

⏱

見積作成に時間がかかる

過去のExcelファイルから類似案件を手動で検索し、単価を比較・調整する作業に膨大な時間を要している。

👤

属人化・ナレッジの散逸

ベテランの暗黙知がExcelファイルに分散しており、体系的な知識として蓄積・共有されていない。

⚠

単価の妥当性検証が困難

提示単価が市場相場や過去実績と乖離していないか、客観的に検証する仕組みがない。

Section 02

プロダクトコンセプト

本アプリは「AIが隣にいる見積作成環境」をコンセプトとし、Excelライクな操作感を維持しながら、LLMエージェントがリアルタイムで作成を支援するデスクトップアプリケーションです。

📊

Excel風見積エディタ

AG Gridベースのスプレッドシート UIで、既存のExcelワークフローを損なわずに編集・入力が可能。

🤖

AIエージェント支援

過去見積の類似検索、単価の自動提案、項目の自動生成、妥当性チェックをLLMが実行。

🔒

ローカルファースト

見積データは社内に留まるセキュアな設計。ファイルアクセスはMCPプロトコルで安全に制御。

📁

ナレッジ蓄積

過去見積をSQLiteに自動蓄積し、建物種別・工種ごとの単価データベースとして活用。

Section 05

AI支援機能の詳細

Claude Agent SDKのTool Use機能により、以下のAI支援をエージェントとして実装します。

🔍

類似案件 自動検索

建物種別・規模・工種をキーに、過去見積DBから類似案件を自動検索。ベクトル検索と条件検索のハイブリッドで精度を確保。

💰

単価 自動提案

過去実績の統計値（平均・中央値・分布）に基づき、各項目の推奨単価を提案。相場からの乖離度も表示。

📝

項目の自動生成

建物種別と工事概要から、必要な見積項目を自動で生成。漏れがちな付帯工事も含めて提案。

✅

妥当性チェック

完成した見積書に対し、単価の異常値検出、項目の過不足、合計金額の妥当性を自動レビュー。

Section 07

開発ロードマップ

Phase 1 — 基盤構築
Tauri環境 + Excel読み込み + AG Grid表示
Tauri 2.0プロジェクトのセットアップ、RustによるExcelパーサの実装、AG Gridでの見積書表示UIを構築。基本的なCRUD操作を実装する。
1.5ヶ月
Phase 2 — ナレッジDB構築
SQLite + 過去見積の取り込み・検索機能
過去見積書をSQLiteに自動インポートする仕組みを構築。建物種別・工種でのフィルタリングと、類似案件の検索機能を実装する。
1ヶ月
Phase 3 — AIエージェント統合
Claude Agent SDK + MCP + Tool実装
Claude Agent SDKを導入し、類似案件検索・単価提案・項目生成・妥当性チェックの各Toolを実装。MCPでローカルデータアクセスを安全に制御する。
2ヶ月
Phase 4 — 実証・改善
社内パイロット運用 + フィードバック改善
実業務でのパイロット運用を開始。ユーザーフィードバックに基づきUI/UXとAI提案精度を継続的に改善する。
1.5ヶ月
Section 08

リスクと対策

⚠ 想定リスク

  • Rustの学習コストによる開発遅延
  • LLM APIのレスポンス遅延がUXを損なう可能性
  • 過去見積データの品質・フォーマットのばらつき
  • API利用コストの増大

✅ 対策

  • フロントエンドはReact/TSで既存知見を活用、Rustは段階的に習得
  • ストリーミング応答 + UIの楽観的更新で体感速度を向上
  • インポート時のバリデーションと正規化パイプラインを構築
  • Prompt Cachingの活用とローカルキャッシュで API呼出しを最適化
